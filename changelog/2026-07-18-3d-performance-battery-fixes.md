### Fixed

- 3D map view (both the flythrough simulation and just panning/rotating/tilting normally) was choppy — every waypoint marker was independently registering its own `map.on("move", ...)` listener to draw a "drop line" down to the ground, recomputing and re-rendering on every single camera movement frame. That visual already existed as a single efficient GPU-rendered line layer elsewhere on the map, so the redundant per-waypoint version — the main source of the slowdown, scaling with waypoint count — was removed outright.
- Buildings and camera frustums toggled their inactive 2D/3D layer via `opacity: 0`, which Mapbox still fully rasterizes even at zero opacity — switched to `visibility: "none"`, which the renderer actually skips, cutting the number of extrusion draws roughly in half for missions with many hand-drawn buildings.
- The remote controller's own battery was never captured at all — it reports its battery as a flat field, while the aircraft nests its own the same reading under a `battery` object, and the code only ever checked the nested shape. The live telemetry HUD now shows the aircraft's and the RC's battery as two separate readings, matching what the RC's own screen shows.
- The telemetry HUD had no way to tell a live reading from a stale one — if the connection dropped silently, it would keep showing the last known battery/position forever with no indication anything was wrong, which read exactly like an incorrect value. It now dims and warns when no update has arrived for 15+ seconds.
